Actividad 18

Analisis de Intrusiones con el Modelo Diamante

Contenido
1. Introduccion 2. Comprension del Modelo 3. Analisis de Evento 4. Relacion con la Kill Chain 5. Hilos de Actividad 6. Conclusion

1. Introduccion

En la actualidad, las organizaciones se enfrentan constantemente a amenazas ciberneticas que pueden comprometer la seguridad de sus sistemas y de la informacion. Por ello, es importante contar con metodologias que permitan identificar y analizar los ataques de manera estructurada.

En esta actividad se aplica el Modelo Diamante de Analisis de Intrusiones, el cual permite relacionar elementos clave como el adversario, la victima, la infraestructura y las capacidades utilizadas durante un incidente. Ademas, se complementa con la Cyber Kill Chain para comprender las diferentes etapas del ataque y detectar posibles patrones de comportamiento del atacante.

El proposito de este analisis es fortalecer la comprension de los eventos de ciberseguridad y desarrollar una vision mas clara sobre como se llevan a cabo las intrusiones dentro de una red informatica.

2. Comprension del Modelo

El Modelo Diamante establece cuatro vertices fundamentales que caracterizan cualquier evento de intrusion:

Modelo Diamante de Intrusiones
Adversario Actor responsable
Capacidad Herramientas y tecnicas
Infraestructura Medio de contacto
Victima Objetivo del ataque
Elemento Descripcion Ejemplo Practico
Adversario Es el actor o entidad responsable de utilizar una capacidad contra una victima. Un grupo de ciberdelincuentes motivado por el beneficio economico que planea una campana de ransomware contra hospitales.
Capacidad Se refiere a las herramientas, tecnicas y tacticas que el adversario utiliza para llevar a cabo el ataque. El uso de un malware de tipo troyano disenado especificamente para evadir la deteccion de antivirus comerciales y cifrar archivos de manera ultra rapida.
Infraestructura Es el medio fisico o logico que el adversario utiliza para establecer contacto con la victima o para entregar sus capacidades. Un servidor de Comando y Control alquilado en la nube que el atacante usa para enviar ordenes al malware instalado en la computadora de la victima.
Victima Es el objetivo del ataque. Puede ser una persona fisica, una organizacion, un sistema especifico o incluso un sector economico completo. El departamento de finanzas de una empresa multinacional que recibe correos electronicos de phishing disenados para robar credenciales de acceso a las cuentas bancarias corporativas.

3. Analisis de Evento

Escenario

Una organizacion detecta comportamiento anomalo en un equipo. El analisis revela:

  1. La victima detecta malware.
  2. El malware contiene dominios de Comando y Control (C2).
  3. Los dominios resuelven a direcciones IP externas.
  4. Logs muestran multiples hosts conectandose a esas IP.
  5. WHOIS/IP revela posible origen del atacante.

Metacaracteristicas del Evento

Metacaracteristica Descripcion del Evento
Marca de tiempo Momento exacto en que los sistemas de monitoreo (SIEM/EDR) disparan la alerta de trafico inusual hacia el exterior, correlacionado con los registros historicos de los logs de los hosts internos involucrados.
Fase Comando y Control. El evento se situa en la etapa de post-explotacion, donde el malware ya residente busca establecer un canal de comunicacion persistente con la infraestructura del atacante para recibir instrucciones.
Resultado Compromiso activo de red. El ataque ha sido exitoso en su comunicacion, la presencia de multiples hosts internos contactando las mismas IPs externas confirma que la amenaza ha logrado propagarse o que el canal de control esta plenamente operativo.
Direccion Saliente. Se observa un flujo de datos que se origina dentro del perimetro de la organizacion y se dirige hacia activos externos remotos, evadiendo potencialmente filtros de seguridad perimetral basicos.
Metodologia El malware emplea nombres de dominio especificos contenidos en su codigo para resolver IPs externas, una tecnica disenada para mantener la conexion incluso si el atacante cambia sus servidores fisicos.
Recursos Codigo malicioso detectado, dominios de C2 identificados en el analisis forense, direcciones IPs externas vinculadas a los dominios y herramientas de inteligencia de amenazas como lo es WHOIS para rastrear al adversario.

Preguntas de Analisis

1. Quien es el adversario en este escenario?

El analisis de WHOIS/IP revela el posible origen del atacante. Se define como el actor externo responsable de controlar el malware y los dominios de Comando y Control (C2) para comprometer la organizacion.

2. Cual es la capacidad utilizada?

La capacidad principal es un malware que ha sido instalado en el equipo de la victima. Este software malicioso cuenta con la funcion especifica de contener y ejecutar consultas a dominios de Comando y Control (C2) para recibir instrucciones externas.

3. Que tipo de infraestructura se emplea?

  • Nombres de dominio especificos
  • Direcciones IP externas
  • Servidores remotos que actuan como puntos de Comando y Control

4. Quien es la victima primaria?

La victima primaria es el host inicial donde la organizacion detecto por primera vez el comportamiento anomalo y la presencia del malware. Aunque, la victima se puede considerar que tambien es la organizacion ya que la red ha sido penetrada.

5. Existe evidencia de movimiento lateral?

Los logs muestran multiples hosts conectandose a esas mismas direcciones IP externas. Esto nos muestra que el compromiso se ha extendido desde el equipo inicial hacia otros dispositivos dentro de la misma red interna.

4. Relacion con la Kill Chain

La Cyber Kill Chain permite mapear los eventos del escenario a las fases del ciclo de ataque:

Evento Fase Kill Chain
Deteccion de malware Instalacion. El malware ya se encuentra dentro del sistema comprometido y ha sido identificado por las herramientas de seguridad.
Envio de phishing Entrega (Delivery). El atacante utiliza correos electronicos maliciosos para distribuir el malware hacia la victima.
Conexion a C2 Comando y Control (C2). El malware establece comunicacion con servidores externos para recibir instrucciones del atacante.
Ejecucion del malware Explotacion / Ejecucion. El codigo malicioso se ejecuta dentro del host comprometido aprovechando vulnerabilidades o interaccion del usuario.

5. Hilos de Actividad

Del escenario tipo APT se identifican los siguientes pasos:

  • Phishing dirigido a administrador
  • Ejecucion de malware
  • Conexion a C2
  • Uso del host comprometido como proxy
  • Ataque a segunda victima

Hilo 1: Compromiso Inicial

  1. El administrador recibe un correo de phishing.
  2. El usuario abre el archivo o enlace malicioso.
  3. El malware se ejecuta en el equipo.
  4. El host comprometido establece conexion con el servidor C2.
  5. El atacante obtiene acceso remoto y control parcial del sistema.

Hilo 2: Movimiento Lateral

  1. El atacante utiliza el equipo comprometido como proxy.
  2. Desde el host inicial se realizan conexiones hacia otros dispositivos internos.
  3. Un segundo equipo de la red es comprometido.
  4. El segundo host comienza tambien a comunicarse con el C2.
  5. Se amplia el alcance del ataque dentro de la organizacion.

Relacion entre ambos hilos

El atacante utiliza la primera victima como punto de apoyo para desplazarse lateralmente dentro de la red corporativa. Este proceso, conocido como pivoting, permite extender el ataque hacia otros sistemas internos utilizando conexiones legitimas desde el host comprometido inicialmente.

6. Conclusion

La presente actividad permitio aplicar el Modelo Diamante de Analisis de Intrusiones como una metodologia efectiva para identificar y correlacionar los elementos involucrados en un incidente de ciberseguridad. A traves del analisis realizado, fue posible reconocer la relacion entre el adversario, las capacidades empleadas, la infraestructura utilizada y las victimas afectadas, proporcionando una vision mas clara sobre el comportamiento del ataque.

Asimismo, la integracion con la Cyber Kill Chain facilito la comprension de las distintas fases del incidente, desde el reconocimiento inicial hasta el establecimiento de comunicacion con servidores de Comando y Control (C2), permitiendo identificar evidencias de propagacion y movimiento lateral dentro de la red.

En conclusion, el uso de estos modelos fortalece el analisis forense y la respuesta ante incidentes, ya que permite estructurar la informacion de manera organizada, detectar patrones de amenaza y apoyar la toma de decisiones para mejorar las estrategias de prevencion, monitoreo y proteccion de la infraestructura tecnologica de una organizacion.
