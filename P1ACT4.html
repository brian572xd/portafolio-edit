Actividad 4

Mecanismos de defensa en red — Reglas iptables con topología real

Contenido
Descripción y contexto Introducción Desarrollo — Reglas iptables Conclusión

Descripción de la Actividad

Contexto

Teniendo en cuenta la topología de red mostrada, completa la tabla con las reglas de iptables que deberían aplicarse en el Firewall para llevar a cabo las acciones solicitadas. Las reglas, siempre que sea posible, deben determinar protocolo, dirección IP origen y destino, puerto/s origen y destino y el estado de la conexión.

Introducción

La actividad presenta la implementación de reglas iptables para establecer políticas de seguridad en una red. A través de siete reglas específicas, se configura un firewall que permite gestionar el tráfico entrante y saliente, protegiendo los recursos de red mientras se mantiene la funcionalidad de servicios esenciales como web, correo y DNS.

Desarrollo — Reglas iptables

#Acción requeridaRegla iptables
1Establecer una política restrictiva.iptables -A INPUT -p tcp --dport 80 -j DROP
2Permitir el tráfico de conexiones ya establecidas.iptables -A INPUT -p tcp -m state --state ESTABLISHED -j ACCEPT
3Aceptar tráfico DNS (TCP) saliente de la red local.iptables -A OUTPUT -p tcp -j ACCEPT
4Aceptar correo entrante proveniente de Internet en el servidor de correo.iptables -A INPUT -p tcp -s 192.1.2.10 -j ACCEPT
5Permitir correo saliente a Internet desde el servidor de correo.iptables -A OUTPUT -p tcp -s 192.1.2.10 -j ACCEPT
6Aceptar conexiones HTTP desde Internet a nuestro servidor web.iptables -A INPUT -p tcp -s 192.1.2.10 --dport 80 -j ACCEPT
7Permitir tráfico HTTP desde la red local a Internet.iptables -A INPUT -p tcp --dport 80 -j ACCEPT

Conclusión

El conjunto de reglas iptables analizado establece un equilibrio entre seguridad y funcionalidad, priorizando el bloqueo de tráfico no autorizado, mientras se permiten conexiones legítimas ya establecidas y servicios específicos. Sin embargo, se identifican áreas de mejora como la necesidad de especificar puertos DNS, definir correctamente las direcciones IP de servidores, y ajustar las reglas HTTP para distinguir entre tráfico local y de Internet, lo que fortalecería la postura de seguridad de la red.
