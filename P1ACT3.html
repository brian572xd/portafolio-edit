Actividad 3

Interpretación y traducción de políticas de filtrado en iptables

Contenido
Introducción Desarrollo Conclusión

Introducción

En la administración de sistemas Linux, la seguridad y el control del tráfico de red son aspectos fundamentales. Iptables es la herramienta de firewall por excelencia que permite gestionar este control, actuando como un filtro de paquetes que opera directamente en el núcleo del sistema. Su funcionamiento se basa en una estructura jerárquica y organizada: cuando un paquete de red llega al sistema, este debe atravesar diferentes niveles de decisión.

Desarrollo

1. Flujo de un paquete en iptables

Cuando un paquete llega al sistema, primero pasa por una tabla, después pasa por una cadena y finalmente se ejecuta una regla/acción.

2. Tablas principales y su propósito

TablaPropósito principalEjemplo de uso
FILTERFiltrado de paquetesBloquear conexiones SSH
NATTraducción de direcciones IPPort Forwarding
MANGLEMedir calidad de servicioCambiar cabeceras
RAWSeguimiento de paquetesPaquetes no inspeccionados
SECURITYMarcar etiquetas de seguridadPaquete autorizado o no autorizado

3. Anatomía de un comando iptables

iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

Este comando define la tabla, cadena, protocolo; el parámetro multiport permite analizar varios puertos y estamos analizando los puertos de origen 80 y 443 y los acepta.

4. Variables y opciones comunes

  • Limitar intentos por minuto: --limit 5/minute
  • Filtrar por IP de origen: -s 192.168.25.0/24
  • Ver solo números, sin DNS: --list -n
  • Ver reglas con contadores: -L -V

5. Análisis de una regla compleja

iptables -A INPUT -i eth0 -p tcp -m multiport --dports 22,80,443   -m state --state NEW,ESTABLISHED -j ACCEPT

Permite el tráfico TCP entrante por interfaz eth0 a los puertos 22, 80 y 443 siempre que sea parte de una conexión nueva o establecida.

6. Reglas de práctica

  • Permitir tráfico HTTP entrante:
    iptables -A INPUT -p tcp --dport 80 -j ACCEPT
  • Permitir todo el tráfico saliente:
    iptables -A OUTPUT -j ACCEPT
  • Permitir SSH solo desde la IP 192.168.1.50:
    iptables -A INPUT -p tcp -s 192.168.1.50 --dport 22 -j ACCEPT
  • Tráfico TCP entrante a puertos 80 y 443 solo si es conexión establecida o relacionada:
    iptables -A INPUT -p tcp -m multiport --dports 80,443 -m state --state ESTABLISHED,RELATED -j ACCEPT
  • Tráfico TCP entrante por eth0 a 22, 80 y 443, solo NEW y ESTABLISHED:
    iptables -A INPUT -p tcp -i eth0 -m multiport --dports 80,22,443 -m state --state NEW,ESTABLISHED -j ACCEPT

Conclusión

El recorrido por los fundamentos de iptables nos confirma su papel como una herramienta indispensable y poderosa para la seguridad perimetral en entornos Linux. Concluimos y comprobamos que con una sola línea de comando puede implementar políticas de seguridad complejas, como permitir tráfico nuevo y establecido a servicios críticos.
